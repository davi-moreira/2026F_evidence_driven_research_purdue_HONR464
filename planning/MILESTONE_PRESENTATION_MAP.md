# MILESTONE_PRESENTATION_MAP — develop → present → submit → revise, per milestone

Every milestone is presented or peer-reviewed in class before (or at) submission.
With 5 students, every student presents every time; formats are deliberately
varied so "presenting" never calcifies into one genre. This map is the
authoritative develop/present/submit/revise ledger the validators check.

Legend: dev = in-class development meetings · pres = presentation/review moment ·
due = submission (11:59 PM unless noted) · rev = revision path (standing policy
in `PROJECT_MILESTONES.md`: eligible milestones revisable within 7 days of
feedback for up to half the lost points).

## Presentation-format variety audit

Formats used: pitch (30s/90s/2-min/3-min), gallery walk, clarifying-question
rounds, written two-line reviews, clinic/consults, workshop swap, red-team,
traceability pair-check, delivery rounds, hot seat, full simulation, public
conference, story round, pair analysis, async board exchange, reproduction
exchange with sign-off, table read, formal defense with cross-examination,
ceremony. **No format repeats in consecutive milestones**, satisfying the
"vary the studio" guardrail in `COURSE_MASTER_PLAN.md` §6.

## Oral-communication ramp (deliberate)

30-sec (M00) → 2-min (M01) → 3-min declared (M03, M04) → 90-sec compressed (M05,
M10) → 60-sec technical (M08) → 4-min walkthrough (M09) → rehearsed 1-min/3-min
(M13) → adversarial Q&A (M14, M15) → **public conference (M16)** → reflective
(M17–M18) → **formal defense (M22)**. Stakes and polish rise monotonically;
format length does not — compression is trained as its own skill.
